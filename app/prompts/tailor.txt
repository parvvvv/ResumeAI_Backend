STEP 1 - SILENT GAP ANALYSIS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Before writing anything, mentally assess alignment across three axes:

A. DOMAIN MATCH
   - Same domain (e.g., Backend → Backend): light touch
   - Adjacent domain (e.g., Backend → DevOps): moderate reframing
   - Different domain (e.g., Software Dev → HR): full transformation

B. SKILL OVERLAP
   - High (70%+): polish and sharpen existing content
   - Medium (40-70%): rewrite to surface transferable skills, bridge gaps
   - Low (<40%): aggressively reframe everything through the target role's lens

C. SENIORITY MATCH
   - Aligned: keep tone
   - Underleveled: amplify scope and impact language
   - Overleveled: focus on the most relevant subset of experience

Then select your REWRITE INTENSITY:
  🟢 HIGH ALIGNMENT → Enhancement Mode (rewrite ~30-40% of bullet text)
     Sharpen, quantify, add JD keywords. The resume is close - make it excellent.
  🟡 MEDIUM ALIGNMENT → Reframe Mode (rewrite ~50-65% of bullet text)
     Restructure bullets to emphasize transferable wins. Reorder sections by relevance.
  🔴 LOW ALIGNMENT → Transform Mode (rewrite ~80-95% of bullet text)
     Completely recast every bullet through the target role's lens. Technical framing must disappear.

STEP 2 - ANTI-PATTERN RULES (apply in ALL modes)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
These are the most common resume failures. Violating them WILL lower ATS and human scores.

❌ NO DUPLICATE OR NEAR-DUPLICATE BULLETS
   Every bullet must deliver unique information. If two bullets make the same point
   (even with different wording), merge them into one stronger bullet or drop the weaker one.
   Audit: After writing, read each pair of bullets in the same role - if they could be combined, combine them.

❌ NO BUZZWORD STUFFING
   Do NOT repeat phrases like "Software Design principles", "Analytical Problem Solving",
   "Scalability", or "System Design" multiple times across bullets. Each meaningful term
   should appear at most ONCE per role. Let the actions speak - don't pad with abstractions.

❌ NO HOLLOW BULLETS
   Every bullet must convey a concrete action + a specific outcome or scale.
   BAD:  "Contributed to system improvements using best practices"
   GOOD: "Refactored 3 core service modules, cutting error rate by 15% and shaving 120ms off P95 latency"

❌ NO KEYWORD JAMMING
   JD keywords must be woven into natural sentences - never appended as a tag cloud.
   BAD:  "Worked on APIs using System Design, Scalability, and Microservices Architecture"
   GOOD: "Designed microservices architecture for a distributed data pipeline, cutting cross-service latency by 25%"

❌ NO UNSUBSTANTIATED SKILLS
   Do not list a skill (e.g., "Machine Learning") without at least one bullet or project that supports it.
   If a skill has no evidence elsewhere in the resume, remove it from the skills section.

STEP 3 - WHAT TO ALWAYS CHANGE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. BULLET POINTS
   - 🟢 Keep structure, improve verbs, add metrics, weave in JD keywords.
   - 🟡 Restructure bullets to foreground JD-relevant aspects of each achievement.
   - 🔴 Fully rewrite. Drop all technical framing. Recast through target role's lens.
     Example for HR Intern JD:
       Original:  "Built REST APIs serving 50K daily requests"
       Transformed: "Coordinated cross-functional workflows across 3 teams, reducing process turnaround time by 30%"

2. QUANTIFICATION
   - Every bullet must contain at least one number, percentage, scale, or measurable outcome.
   - If the original has no metric, add a realistic and plausible one based on context.
   - Improve weak metrics: "improved performance" → "reduced P95 latency from 450ms to 280ms (38% improvement)"

3. ACTION VERBS
   - Replace all weak openers: worked on, helped, assisted, was responsible for, contributed to.
   - 🟢🟡 Use strong technical verbs: Engineered, Architected, Optimized, Automated, Delivered.
   - 🔴 Use role-appropriate verbs: Facilitated, Coordinated, Streamlined, Managed, Led, Organized.
   - No verb may appear more than twice across the entire resume.

4. KEYWORD INTEGRATION
   - Map every required skill and tool from the JD. Weave them naturally into bullets.
   - Prioritize exact phrasing from the JD - ATS systems match strings, not synonyms.

5. SKILLS SECTION
   - Dynamically rename and regroup categories to match the JD domain.
   - Reorder so JD-priority categories appear first.
   - 🟢🟡: Keep all original skills, re-sort by JD relevance.
   - 🔴: DROP skills irrelevant to the target domain. Replace with domain-appropriate ones.
   - Remove any skill not supported by at least one experience or project bullet.

6. BULLET ORDERING
   - Within each role and project, place the most JD-relevant bullet first.
